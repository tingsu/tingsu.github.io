A bibliography of papers related to data flow testing

Overview

Data flow testing (DFT), introduced by Herman in 1976, is a family of testing strategies based on selecting paths from the program’s control flow in order to explore sequences of events related to the status of variables or data objects. It fills the gaps between all path testing and branch/statement testing with the aim to pinpoint the potential data-flow anomalies. The motivation behind is to verify the correctness of defined variable values by observing their corresponding uses can produce the desired results. See wiki for details about white-box testing techniques.

Various approaches have been used to automate data flow testing, for example the following approaches have been used in data flow-based test generation:

  • Random Testing: random or adaptive random test generation.
  • Collateral Coverage-based Testing: exploit the subsumption relation between test objectives to reduce testing cost and infer coverage.
  • Search-based Testing: search algorithms (e.g. genetic algorithms, optimization algorithms) are used guided by a fitness function to sastify test objectives.
  • Symbolic Execution-based Testing: it includes traditional (i.e. static) symbolic execution and dynamic symbolic execution.
  • Model Checking-based Testing: a verfication-based testing method used to generate conterexamples as tests.
